内网服务器暴露到外网（linux下的Apache2端口修改）

前话

说实话，做完了，其实很简单。做的时候还是遇到了一些问题的。这就是经验的积累了。我之前做运维的时候主要是做windows平台的，运维的项目也有很多个，其中还有企鹅的服务器。这次做linux的过程中，还是很不熟练。主要遇到的问题主要有两个：一个是企业路由器的设置（80端口的限制）：以前上学的时候都是个人路由器，工作时有专门的IT服务部以及云计算部门，这些事情都不需要我们开发做；一个是Apache2服务器下端口的更改，linux命令不熟，以前主要tomcat等，apache2真不熟。

路由器的设置

路由器的设置，其实大同小异，熟悉就好了。按下面的步骤操作。

  • 首先设置内网要暴露的ip及端口。

  • 在DMZ中设置内网要暴露的ip。

  • 绑定动态域名。自己去花生壳注册吧。

理论上来讲，做好这几步其实就已经把机器暴露出去了。如果你部署的网站或应用是80以外的端口，后面的便没什么要做的了。但是如果你的网站端口为80，那就继续往下看咯！（80端口是因为很多服务商做了限制，所以没法开放出来，所以只好去换应用的端口咯！）

Apache2端口的修改

  • 找到apache2文件夹，可以通过find命令寻找 sudo find / -name ‘apache2’
  • 或者找到ports.conf文件编辑，sudo vi /etc/apache2/ports.conf,修改80为8088（自己选择端口）
  • 不要以为只需要修改这一个地方，还有一个地方。/etc/apache2/sites-enabled/000-default.conf,同样改为8088

结尾

大功告成！什么时候能跟着自己的心走！

Hexo博客的搭建

搞这个看了很多攻略，反正成功是很多失败铺垫起来滴！好，开干！

node.js安装

  • 下载地址：https://nodejs.org/
  • 说明：这个自己安装吧,很简单。

hexo本地安装

  • 1.建立文件夹,自己记住路径

  • 2.进入myblog文件夹中，按住shift，点击鼠标右键，在当前窗口打开命令行

  • 3.命令行中依次运行 (1).hexo init (2).npm install

  • 4.命令行继续运行 (3).hexo server （如果运行不成功，请运行npm install hexo-deployer-git —save）

  • 5.在浏览器中输入localhost:4000

github创建repository

(自己安装github的windows平台gitshell,并配置好环境变量！！！，切记)

  • 1.进入github，申请账号， https://github.com/

  • 2.配置git的SSH库，在gitshell中运行命令
    ssh-keygen -t rsa -C "邮件地址@youremail.com"
    当出现Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>
    之后会要求输入用户名密码，输入即可。记住文件保存地址，待会要用。

  • 3.申请完后创建新的repository,并添加SSH Key到GitHub

  • 4.找到2中保存的pub文件，复制里面的密钥。

部署hexo到github

  • 1.修改hexotest/myblog中的_config.yml文的deploy一项
    deploy: 

    type: git
repo: https://github.com/xxx/xxx.github.io.git
branch: master

  • 2.使用gitshell进入hexotest/myblog陆续执行下列命令
    hexo clean
    hexo generate
    hexo deploy

  • 3.浏览器输入xxx.github.io即可访问，XXX为你自己设置的名字。

  • 4.更新内容可到网上搜索hexo的相关设置，这个网上资源很多，自己百度谷歌。
